Transliteration

Converting one script to another

Transliteration is not transcription

Mainly concerned with preserving spelling

Transliteration !== transcription It's mainly concerned with preserving the spelling, rather than pronunciation e.g. Arabic > English, Yiddish > English Generally the aim is to preserve the pronunciation. LOC specifies that Author names are transliterated by country of origin Yiddish authors come from all over the world, so this is complicated This is why we had to fall back on empirical analysis Otherwise we'd have a HUGE number of characters to support There are tables to guide transliterators on how to transliterate each script So we can just look at the tables for Yiddish, right? Useful to make it searchable and readable by people who don't read Yiddish script

Some thoughts on BEM and CSS

A lightning talk by Jon White / @superbland

CSS is really good when...

  1. Make changes without breaking stuff
  2. You can easily tell what a class does
  3. Figuring out what does what is pretty easy
  4. I write some new styles and it works

CSS is not so good when...

  1. Changes that solve one problem create another
  2. Unnecessary weight in the inheritance chain
  3. Developers who prefer brevity over clarity
  4. I need to resort to !important; rules or concoct clever inheritance hacks
